# 零 -ZERO- MEMBERS パスワード保護・月次更新マニュアル

本サイト「零 -ZERO- MEMBERS」は、GitHub Pages上で公開されている会員限定ポータルサイトです。退会者等による不正閲覧を防ぐため、StatiCrypt（AES-256）によるページごとの暗号化保護を導入しています。

毎月15日の合言葉（パスワード）更新時は、以下の**3ステップ**で一括暗号化およびデプロイを行ってください。

---

## 毎月の更新手順（3ステップ）

### ステップ1：パスワードを指定して一括暗号化を実行

ターミナルを開き、プロジェクトのルートディレクトリで新しい合言葉（パスワード）を引数に指定してコマンドを実行します。

```bash
./encrypt.sh "新しい合言葉"
```
または、npmスクリプトからも同様に実行可能です：
```bash
npm run encrypt -- "新しい合言葉"
```

> [!NOTE]
> `private_html/` フォルダ内の平文HTML（全30ページ）が読み込まれ、AES-256で暗号化された公開用HTMLファイルがルートディレクトリに自動出力されます。

---

### ステップ2：ローカルで動作確認

ブラウザでルートディレクトリの `index.html` を開きます。
1. 新しい合言葉を入力し、「認証してページを開く」をクリックしてログインできるか確認します。
2. 「合言葉を31日間記憶する (Remember me)」にチェックが入っていること、およびログイン後に講義ページへスムーズに遷移できるか確認します。
3. サポートポータル（`support.html`）および質問フォーム（`question-form.html`）が、パスワードなしで平文閲覧できるか併せて確認します。

---

### ステップ3：GitHubへプッシュしてデプロイ

変更をGitステージに登録・コミットし、リモートリポジトリ（GitHub）へプッシュします。

```bash
git add .
git commit -m "Update monthly password to [適用月]"
git push
```

プッシュ完了後、1〜2分程度でGitHub Pagesに自動デプロイされ、サイト全体が新しい合言葉で保護されます。

---

## システム仕様・ディレクトリ構成

```
零コミュニティ（有料）/
 ├── private_html/          # 【重要・非公開】平文のHTMLソース (全30ページ)
 ├── scripts/               # 暗号化スクリプトエンジン (Python Fallback 等)
 ├── assets/                # 画像・サムネイル等
 ├── style.css              # 共通デザインスタイルシート
 ├── template_zero.html     # パスワード入力画面のカスタムテンプレート (ゴールド調)
 ├── encrypt.sh             # 一括暗号化シェルスクリプト
 ├── package.json           # npm 構成ファイル
 ├── .gitignore             # private_html/ や node_modules/ を除外する設定
 ├── index.html             # 【公開】暗号化済みのトップページ
 ├── archive-*.html         # 【公開】暗号化済みの月別アーカイブページ
 ├── article-*.html         # 【公開】暗号化済みの有料会員向け記事ページ (26本)
 ├── zoom.html              # 【公開】暗号化済みのZOOM講義ページ
 ├── support.html           # 【公開】無料公開ページ (サポート・退会申請)
 └── question-form.html     # 【公開】無料公開ページ (質問・相談フォーム)
```

### セキュリティと保護設計
- **平文ソースの保護**：`.gitignore` により `private_html/` ディレクトリは Git 追跡から完全に除外されており、GitHub リポジトリ上で平文コードが公開されることはありません。
- **クライアントサイド暗号化 (AES-256)**：JavaScript による単なる見せかけのパスワード判定ではなく、暗号化コンテンツ本体（暗号文）のみが公開されるため、ソースコードを閲覧してもパスワードや本文を解読することは不可能です。
- **Remember me 機能の整合性**：月次の合言葉変更と整合させるため、ブラウザ記憶の有効期限は最大31日間に設定されています。
